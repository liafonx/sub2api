## Key Patterns

### Account `Extra` field
All optional account settings live in `Account.Extra` (JSONB). Add new settings by:
1. Writing an accessor method on `Account` in `account.go` (`GetFoo()`, `IsFooEnabled()`)
2. Building/reading the key in frontend modal `extra` construction blocks
3. Mapping in `handler/dto/mappers.go` if the field needs to be in API responses

### Service interfaces
Interfaces live in `internal/service/`, Redis implementations in `internal/repository/`. The pattern:
```
service/foo.go          → type FooCache interface { … }
repository/foo_cache.go → type fooCache struct { rdb *redis.Client }
                          func NewFooCache(rdb) service.FooCache
```

### Dependency wiring
`cmd/server/wire_gen.go` is **manually maintained** (not generated by Wire). New dependencies are added by hand in the `InitializeApp` function. Use `SetFoo()` setter methods on services for optional post-construction wiring.

### Fail-open principle
All Redis-backed scheduling checks (`isAccountSchedulableForWindowCost`, `CheckUserQuotaForAccount`, etc.) return `true`/allowed on Redis errors to prevent outages.

## Plugins & Features

### Window Cost Limit (upstream)
Tracks 5-hour rolling cost per Anthropic account. Three-zone model (green/yellow/red) controls scheduling. Config in `account.Extra["window_cost_limit"]`.

### Session Count Limit (upstream)
Per-account max concurrent sessions tracked in Redis sorted set. Config in `account.Extra["max_sessions"]`.

### RPM Limit (upstream)
Per-account requests-per-minute cap. Tiered or sticky-exempt strategy. Config in `account.Extra["base_rpm"]`.

See [`ACTIVE_PATCHES.md`](ACTIVE_PATCHES.md) for full details on all fork patches (per-user quota, peak usage, dynamic cost, affinity, RPM, etc.).

## Build & Deploy (summary)

Full details in `deploy/AGENTS.md` (gitignored, local dev machine only). Quick reference:

```bash
# 1. Build frontend (always first — backend embeds it)
cd frontend && pnpm run build

# 2. Cross-compile for Mac Mini (x86_64)
cd ../backend
CGO_ENABLED=0 GOOS=darwin GOARCH=amd64 go build -tags embed -ldflags "-s -w" -o sub2api ./cmd/server

# 3. Deploy
scp -o ServerAliveInterval=10 -o ServerAliveCountMax=2 backend/sub2api liafonx@Liafonxs-Mac-mini.local:/tmp/sub2api
ssh liafonx@Liafonxs-Mac-mini.local 'launchctl bootout gui/$(id -u)/com.sub2api; cp /tmp/sub2api /usr/local/bin/sub2api; launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.sub2api.plist && tail -20 /usr/local/var/log/sub2api/sub2api.log'
```
